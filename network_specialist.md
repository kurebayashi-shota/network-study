# DNS

- 名前解決=UDP:53,ゾーン転送=TCP:53
- 再帰問い合わせ: クライアント->キャッシュ
- 反復問い合わせ: キャッシュ->ルート->TLD 権威->権威
- URL(スキーマ+FQDN(ホスト+ドメイン))
- ゾーン転送:権威 DNS がマスターとスレーブで同期
  ゾーン情報に更新があった
  [マスター]--notify:UDP-->[スレーブ]
  [マスター]<--下さい:TCP--[スレーブ]
  SOA レコードのリフレッシュ間隔時間に更新がなかった場合
  [マスター]<--更新ない?--[スレーブ]
  [マスター]--ないよ-->[スレーブ]

| A    | AAAA | MX     | CNAME | NS     | PTR    | TXT  |
| ---- | ---- | ------ | ----- | ------ | ------ | ---- |
| IPv4 | IPv6 | メール | 別名  | ゾーン | 逆引き | 自由 |

- DKIM,DMARK,SPF レコードは TXT で管理
- SOA レコード:マスター(ゾーン管理)とスレーブ(差分)で管理内容が違う

## EAP-TLS フロー

1. **EAPOL 開始**
   [Supplicant] ------------------> [Authenticator]

- 認証したいのでサーバーさんへ通してください
  (ネットワークアクセス開始)

2. **EAP 要求/Identity**
   [Supplicant] <------------------ [Authenticator]

- ID 教えて
  (サプリカントに ID 要求)

3. **EAP 応答/Identity**
   [Supplicant] ------------------> [Authenticator]

- 私の ID です。  
  (ユーザー名/端末 ID 送信)

4. **RADIUS Access-Request (EAP Identity)**
   [Authenticator] ------------------> [AuthenticationServer]

- サーバーさんこの ID の認証方式を教えて
  (EAP Identity を RADIUS で転送)

5. **RADIUS Access-Challenge (EAP-TLS)**
   [Authenticator] <------------------ [AuthenticationServer]

- TLS です
  (TLS ベースの認証方式指示)

6. **証明書をどうぞ:往復**
   [Supplicant] <--> [Authenticator] <--> [AuthenticationServer]

- サーバ証明書提示
- サプリカント証明書送信

7. **PMK 生成用の乱数をどうぞ:往復**
   [Supplicant] <--> [Authenticator] <--> [AuthenticationServer]

- それを基に互いに PTK(共通鍵)を生成
- IKEv1 に似てる

8. **EAP-TLS Handshake**
   [Supplicant] <--> [Authenticator] <--> [AuthenticationServer]

- 4way ハンドシェイク(TLS ハンドシェイク開始)

  - 鍵交換・暗号化トンネル確立

9. **EAP-TLS Success / RADIUS Access-Accept**
   [Supplicant] <-- [Authenticator] <-- [AuthenticationServer]

- 鍵作ったから通っていいよ  
   (認証成功、ネットワークアクセス許可)

10. **ネットワークアクセス開始**  
    [Supplicant] ------------------> ネットワーク

# 攻撃

- DHCP スヌーピング:DHCP をのぞき見る
- DHCP キャッシュポイズニング:DHCP キャッシュ情報を偽 IP に変更させる
- カミンスキー攻撃:存在しないドメインの名前解決を要求して ↑ をする
- ARP スプーフィング: ARP テーブルを汚染してこっそり中間者になる。
  通信制御装置を中間者にして防御する事も出来る
- IP スプーフィング:送信元 IP を偽装する
- DNSamp 攻撃:IP スプーフィングをして bot ネットに DNS 問い合わせさせる
- スマーフ攻撃:IP スプーフィングをしてブロードキャストで ICMP
- プードル攻撃:SSL3 の脆弱性をつく攻撃
- ビースト攻撃:TLS1.1 の脆弱性をつく攻撃
- RLO 攻撃:右から左にオーバーライドして偽装
- Syn フラッド攻撃:Syn を送り続けて Syn/Ack に対して Ack を返さない
- mirai:IoT に感染して DDoS 攻撃させる
